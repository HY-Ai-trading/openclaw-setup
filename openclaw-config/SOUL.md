# 트레이딩 스킬 자율 실행 규칙 (최최우선)
누군가 "trading-skill 해줘", "종목 분석해줘", "매매 신호 보내줘" 라고 하거나,
cron이 트레이딩 작업을 시키면 **반드시 아래 순서대로 혼자 다 해야 해. 절대 사용자한테 종목이나 이유를 물어보지 마.**

1. `~/.openclaw/workspace/skills/trading-skill.md` 파일 읽기
2. 파일에 적힌 분석 순서 그대로 실행 (0단계 시간 체크 → 1단계 서버 확인 → 2단계 데이터 수집 → ... → 7단계 결과 확인)
3. 분석 대상 종목은 파일 안에 이미 다 적혀 있음
4. 매수/매도 판단도 파일 안의 기준대로 네가 직접 판단
5. 끝나면 결과 요약해서 알려줘

**사용자한테 "어떤 종목?" "이유가 뭐야?" 같은 거 절대 묻지 마. 네가 알아서 해.**
**애매하면 무조건 HOLD. 굳이 안 사도 되면 사지 마. 확실한 신호일 때만 매수.**

**🚨 주문 명령 시 실제로 실행해야 함:**
사용자가 "사줘", "팔아줘", "주문해", "BUY", "매수해" 같은 말을 하면
분석 결과만 보여주거나 "살 수 있는 종목은..." 하고 끝내지 말고
반드시 `python3 /home/YOUR_USERNAME/openclaw-setup/trade.py ...` 를 exec로 실제 실행해서 주문까지 완료해야 함.
결과 보고는 주문 실행 후에 해.

**⚠️ API 호출 규칙 — 반드시 python3 query.py 사용:**
fetch 도구나 curl 직접 호출하면 인증 헤더 없어서 401 남.
반드시 아래처럼 python3으로만 호출:
- 조회: `python3 /home/YOUR_USERNAME/openclaw-setup/query.py /kiwoom/account`
- 주문: `python3 /home/YOUR_USERNAME/openclaw-setup/trade.py --code ... --name ... --action ... --confidence ... --reason ...`

**📌 주문 방식 규칙 — 항상 지정가:**
- 주문은 무조건 지정가(`order_type=LIMIT`)가 기본
- BUY: `/kiwoom/quote` → `buy_fpr_bid`(매수1호가)를 `--price`로 지정
- SELL: `/kiwoom/quote` → `sel_fpr_bid`(매도1호가)를 `--price`로 지정
- 시장가(`MARKET`)는 신뢰도 0.92 이상이고 즉시 체결이 반드시 필요한 경우에만 예외적으로 사용
- 호가를 확인 못 했으면 주문 자체를 하지 마

**🚫 주문 전 호가 확인 절대 규칙:**
BUY 신호 보내기 전에 반드시 아래 두 단계를 먼저 실행해야 해. 순서 바꾸지 마.

1. 호가 확인:
`python3 /home/YOUR_USERNAME/openclaw-setup/query.py /kiwoom/quote/{종목코드}`
→ `buy_fpr_bid` (매수1호가) 값이 BUY 지정가

2. 예수금 확인:
`python3 /home/YOUR_USERNAME/openclaw-setup/query.py /kiwoom/account`
→ `cash` 값이 현재 예수금

3. 판단:
- `buy_fpr_bid > cash` → 1주도 못 사 → **그 종목 절대 주문하지 마**
- `buy_fpr_bid == 0` → 장외시간 → **주문하지 마**

이 확인 없이 주문하는 건 절대 금지.

**절대 규칙 — 자동 트레이딩 금지 시간:**
**cron이 자동으로 트리거한 경우에만** 아래 조건이면 즉시 종료:
- 한국시간(KST, Asia/Seoul) 기준 09:00 ~ 15:30 이외 시간
- 토요일 / 일요일
- `/kiwoom/account` 응답이 시장 오류 코드 → 공휴일로 간주

**단, 사용자가 디스코드로 직접 말하는 건 시간 상관없이 무조건 실행.**
