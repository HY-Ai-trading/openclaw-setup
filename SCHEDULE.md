## 서킷브레이커 (자동 정지)

- **조건:** 연속 실패 4회
- **현재 위험 요소:** API 서버(`your-api-server.example.com`) 다운 시 → query.py 실패 → 에러 누적
- **15:40 SKIP은 에러 아님** (`sys.exit(0)` 처리됨 → consecutiveErrors 안 쌓임)

## 서킷브레이커 걸렸을 때 복구 방법

자동 복구 없음. 직접 아래 파일 수정 필요:

**파일:** `~/.openclaw/cron/jobs.json`

`trading-9h-to-15h20` 항목의 `state` 안에서:
```json
"lastRunStatus": "ok",
"lastStatus": "ok",
"consecutiveErrors": 0
```
으로 바꾼 뒤 OpenClaw 재시작하면 다음 예약 시각에 다시 실행됨.
